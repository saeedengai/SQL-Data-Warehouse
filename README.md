# SQL Data Warehouse & Analytics Project

Welcome to the **Data Warehouse & Analytics Project** repository! 🚀  
This portfolio project walks through an end-to-end data warehousing and analytics solution—from standing up a warehouse to producing actionable insights—while showcasing best practices in data engineering and analytics.

---

## 🚀 Project Requirements

### Building the Data Warehouse (Data Engineering)

**Goal**  
Create a modern data warehouse in **SQL Server** that consolidates sales data to support analytical reporting and informed decisions.

**Specifications**  
- **Data Sources:** Load data from two systems (**ERP** and **CRM**) provided as CSVs.  
- **Data Quality:** Clean and fix quality issues before analysis.  
- **Integration:** Merge both sources into one intuitive, analytics-ready data model.  
- **Scope:** Work with the most recent dataset only; historical storage isn’t required.  
- **Documentation:** Deliver clear documentation of the data model for both business and analytics users.

---

### BI: Analytics & Reporting (Data Analytics)

**Goal**  
Build **SQL-driven** analyses to provide deep insights into:
- **Customer behavior**
- **Product performance**
- **Sales trends**

These outputs equip stakeholders with key business metrics to drive **strategic decision-making**.
